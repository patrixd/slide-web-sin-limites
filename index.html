CSS

Prioridades

Prioridad Tipo de origen de CSS Descripción
1 Importancia La anotación ‘!important’ sobreescribe la prioridad anterior
2 Inline Un estilo aplicado a un elemento HTML por medio del atributo 'style'
3 Media Type Una propiedad aplica a todas las media types, a menos que un media type específico CSS esté definido
4 Definido por el usuario La mayoría de los navegadores tienen esta característica de accesibilidad: un estilo CSS definido por el usuario
5 Especificidad del selector Un selector contextual específico (#heading p) sobreescribe una definicón general (p)
6 Orden de las reglas La última regla especificada tiene una mayor prioridad
7 Herencia Si una propiedad no está especificada, es heredada del elemento padre
8 Definición de propiedad CSS en el documento HTML Una regla CSS común sobreescribe el valor del navegador
9 Por defecto del navegador La prioridad más baja: estos valores son determinados por las especificaciones iniciales de la W3C
- Hay propiedades que se heredan en CSS y otras que no. Ejemplo, el color se hereda de etiquetas padres a hijos. - Cuando no se especifica un valor para un elemento de una propiedad heredable, el elemento toma el valor calculado de la propiedad de su elemento padre. Sólo los elementos raíz de un documento obtienen el valor inicial dado en el resumen de la propiedad. - Si no se especifica un valor para una propiedad no heredable, el elemento toma el valor inicial de dicha propiedad. Ejemplo, borde no se hereda y toma por defecto el valor que indica que no hay borde.

CSS

Propiedades

- CSS se ha creado en varios niveles y perfiles.
- Cada nivel de CSS se construye sobre el anterior, generalmente añadiendo funciones.
 - Los perfiles son parte de uno o varios niveles de CSS definidos para un dispositivo o interfaz particular.
- A diferencia de CSS2 y CSS1, CSS3 se ha dividido en módulos. Teniendo como ventaja una aceptación más rápida que en las especificaciones anteriores. CSS2 tardó 4 años en tener una revisión oficial (CSS2.1 en 2011 desde el 2008), y ahora están trabajando en la revisión 2.
- Cada módulo de CSS3 añade nuevas funcionalidades a las definidas en CSS2, de manera que se conservan las anteriores manteniendo la compatibilidad.

DOM

Document Object Model

- Modelo de Objetos del Documento
- A través del DOM, los programas pueden acceder y modificar el contenido, estructura y estilo de los documentos HTML y XML.
- El responsable del DOM es el World Wide Web Consortium (W3C).


- Imagen: https://www.mediaevent.de/javascript/DOM.html

DOM

Usado para visualizar las páginas web por el navegador

- Cuando el navegador muestra una página web:
- Carga los elementos HTML en el DOM en memoria y los combina con sus estilos CSS.
- Finalmente la visualiza.


- Imagen: https://developer.mozilla.org/en-US/docs/Learn/CSS/Introduction_to_CSS/How_CSS_works

- Podemos modificar el DOM gracias al lenguaje de programación Javascript.

Responsive Web Design

RWD

- Técnica que permite hacer páginas Web adaptables en función de la ventana de visualización (viewport) del dispositivo.
- Dispositivos como TV, escritorio, móviles, tablets, ebooks, relojes...
-

Responsive Web Design

Página flexible con min-width y unidades relativas (%)

- Antes sólo teníamos diseños fluidos que permitían usar todo el ancho de la pantalla, haciendo uso de porcentajes (unidad relativa al contenedor) en lugar de pixeles (unidad absoluta). Pero aún así no eran fáciles de usar. Textos muy largos en televisiones, columnas laterales muy estrechas en móviles.

Responsive Web Design

Página responsive con max-width y media-queries

- Pero CSS3 nos proporcionó el uso de media queries y propiedades como max-width, que nos permiten adaptarnos a la ventana de visualización mediante CSS y HTML, y al uso de imagenes diferentes para diferentes resoluciones para no sobrecargar a los usuarios móviles.

Responsive Web Design

- El 57% de los usuarios abandonan las páginas si tardan más de 3 segundos en cargar.
- Y Google no penaliza a las páginas optimizadas para móviles en sus buscadores.

Responsive Web Design

¿Cómo se hace?

  1. Ajustar ancho de la página al ancho del dispositivo 
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
  2. Añadir media queries para añadir estilos condicionales. 
    @media only screen and (max-width : 768px) {
- Ajustamos el ancho de la página al ancho del dispositivo.
- Porque la mayoría de smartphones simulan tamaños de pantalla mayores, haciendo una especie de dimensiones virtuales que faciliten la lectura de webs que no están diseñadas para Responsive Web Design.
- Las Media Queries son, consultas de las características del medio donde se está visualizando una web y nos sirven para definir estilos condicionales, que solo se aplicarán en caso que esa consulta del medio sea satisfactoria.
- En este caso la query selecciona los pequeños dispositivos y tablets
